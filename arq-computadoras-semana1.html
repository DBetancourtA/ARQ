// Módulo 1 — Contexto
Introducción general
Hardware Software Firmware Sistema computacional
▼

La arquitectura de computadoras es la ciencia que estudia cómo se diseñan, organizan y conectan los componentes de un sistema computacional para que puedan procesar información de manera eficiente. Es el "plano" que define cómo funciona una computadora por dentro.

// Los tres niveles de un sistema computacional
🔩
Hardware
Componentes físicos y tangibles: procesador, memoria RAM, disco duro, tarjeta gráfica, placa madre. Todo lo que puedes tocar.
💻
Software
Programas e instrucciones que le dicen al hardware qué hacer. Incluye el sistema operativo, aplicaciones y código Python que ya conoces.
⚡
Firmware
Software de bajo nivel grabado directamente en un chip de hardware. Intermedio entre los dos: el BIOS de tu PC es firmware. No se puede modificar fácilmente.
💡
Analogía clave: el hardware es el cuerpo, el software es la mente, y el firmware es el instinto — respuestas básicas programadas que el sistema necesita para arrancar.
// ¿Qué es un sistema computacional?

Un sistema computacional es el conjunto integrado de hardware, software y firmware que trabajan juntos para recibir entrada, procesar información y producir una salida. No es solo la CPU — es el ecosistema completo.

ComponenteRolEjemplo
EntradaRecibe datos del usuario o del entornoTeclado, mouse, sensores
ProcesamientoTransforma y calcula con los datosCPU, GPU
AlmacenamientoGuarda datos e instruccionesRAM, SSD, HDD
SalidaPresenta los resultados al usuarioMonitor, impresora, altavoces
ComunicaciónConecta el sistema con otrosTarjeta de red, Bluetooth

// Módulo 2 — Historia
Evolución de las computadoras
Generaciones Tubos de vacío Transistores Microprocesadores
▼

La historia de las computadoras se divide en cinco generaciones, cada una marcada por un salto tecnológico que redujo el tamaño, aumentó la velocidad y bajó el costo drásticamente. Esta historia justifica por qué la arquitectura actual es como es.

1ª Generación
Tubos de vacío
Computadoras del tamaño de una habitación. Generaban calor extremo y fallaban constantemente. La ENIAC usaba 18,000 tubos.
1940 — 1956
2ª Generación
Transistores
Reemplazaron los tubos: más pequeños, más rápidos, menos calor. Surgieron los primeros lenguajes de programación (COBOL, FORTRAN).
1956 — 1963
3ª Generación
Circuitos integrados (IC)
Miles de transistores en un solo chip de silicio. Aparecen los sistemas operativos y el concepto de "tiempo compartido".
1964 — 1971
4ª Generación
Microprocesadores
Toda la CPU en un solo chip (Intel 4004, 1971). Nace la computadora personal (PC). La Ley de Moore predice que los transistores se duplican cada 2 años.
1971 — 2010
5ª Generación
IA, computación cuántica, microcontroladores
Arquitecturas especializadas para IA (TPU de Google, GPU de NVIDIA). IoT, computación en la nube y chips integrados en todo dispositivo.
2010 — Presente
📐
Ley de Moore (1965): el número de transistores en un chip se duplica aproximadamente cada dos años, manteniendo el mismo costo. Esta tendencia ha impulsado el avance tecnológico durante décadas, aunque hoy está llegando a sus límites físicos.

// Módulo 3
El Procesador (CPU)
Central Processing Unit Ciclo Fetch-Decode-Execute Velocidad de reloj
▼

El procesador o CPU (Central Processing Unit) es el cerebro del sistema computacional. Se encarga de ejecutar instrucciones: leer un dato, sumar dos números, comparar valores, escribir un resultado. Todo lo que un programa hace, pasa por la CPU.

⏱️
Velocidad de reloj
Medida en GHz (gigahercios). Un procesador a 3 GHz ejecuta 3,000 millones de ciclos por segundo. Cada ciclo puede procesar una o más instrucciones.
🔢
Núcleos (cores)
Un procesador moderno tiene múltiples núcleos (2, 4, 8, 16+). Cada núcleo es una CPU independiente que puede ejecutar tareas en paralelo.
📏
Tamaño del proceso
Se mide en nanómetros (nm). Un chip de 5nm tiene transistores de 5 millonésimas de milímetro. Mientras más pequeño, más eficiente y rápido.
// El ciclo de instrucción: Fetch → Decode → Execute
💡
Fetch: la CPU busca la instrucción en memoria RAM. Decode: la Unidad de Control descifra qué operación realizar. Execute: la ALU o el hardware especializado ejecuta la operación. Este ciclo se repite miles de millones de veces por segundo.
04
// Módulo 4
Partes del Procesador
CUALU RegistrosCaché
▼
🎛️
Unidad de Control (CU)
Dirige el tráfico dentro del procesador. Busca instrucciones en memoria, las decodifica y le dice a la ALU qué operación realizar y con qué datos.
🧮
Unidad Aritmético-Lógica (ALU)
Ejecuta todas las operaciones matemáticas (suma, resta, multiplicación, división) y lógicas (AND, OR, NOT, comparaciones). Es la "calculadora" del CPU.
📌
Registros
Memoria interna del procesador, ultrarrápida pero muy pequeña (bytes). Guardan temporalmente los datos que la CPU está usando en este preciso instante.
⚡
Memoria Caché
Memoria rápida entre la CPU y la RAM. Guarda copias de los datos más usados para no tener que ir siempre a la RAM más lenta. Hay niveles: L1, L2, L3.
⚠️
Registros clave: el PC (Program Counter) apunta siempre a la próxima instrucción a ejecutar. El IR (Instruction Register) guarda la instrucción actual. El Acumulador almacena el resultado de la ALU.

// Módulo 5
Buses del Sistema
Bus de datos Bus de direcciones Bus de control
▼

Los buses son las "autopistas" de la computadora — canales por donde viajan los datos entre la CPU, la memoria y los periféricos. Sin buses, los componentes no pueden comunicarse.

BusFunciónDirecciónEjemplo
Datos Transporta los datos reales (bits) entre componentes Bidireccional El contenido de una variable
Direcciones Indica dónde en memoria leer o escribir Unidireccional (CPU → Memoria) La dirección 0x0042 en RAM
Control Envía señales de comando (leer, escribir, reset) Bidireccional Señal READ o WRITE
💡
Ancho del bus: un bus de 32 bits puede transportar 32 bits a la vez; uno de 64 bits, el doble. Por eso los sistemas operativos de 64 bits son más eficientes que los de 32 bits — el "ancho de la autopista" es el doble.

// Módulo 7
Arquitectura Von Neumann
Memoria única John von Neumann 1945
▼

Propuesta por John von Neumann en 1945, esta arquitectura define que los datos y las instrucciones comparten la misma memoria y el mismo bus. Es el modelo sobre el que se construyeron casi todas las computadoras de propósito general que existen hoy.

⚠️
El cuello de botella de Von Neumann: como datos e instrucciones comparten el mismo bus, la CPU debe esperar a que uno termine para acceder al otro. Esta limitación se llama "Von Neumann bottleneck" y fue el principal problema que motivó la arquitectura Harvard.

// Módulo 8
Arquitectura Harvard
Memorias separadas Buses paralelos Microcontroladores
▼

La arquitectura Harvard separa físicamente la memoria de instrucciones y la memoria de datos, cada una con su propio bus. Esto permite acceder a ambas al mismo tiempo, eliminando el cuello de botella de Von Neumann.

CaracterísticaVon NeumannHarvard
MemoriaUna sola (datos + instrucciones)Separada (dos memorias)
BusesUn bus compartidoBuses independientes
VelocidadCuello de botellaAcceso paralelo — más rápido
ComplejidadSimple — fácil de implementarMás compleja
Uso típicoPCs, servidores, laptopsMicrocontroladores (PIC, AVR, ARM)
💡
Harvard modificada: los procesadores modernos (Intel, AMD) usan una variante llamada Harvard modificada — internamente separan las cachés de instrucciones y datos (L1-I y L1-D), pero externamente comparten la memoria RAM. Obtienen lo mejor de ambos mundos.

// Módulo 9
Arquitectura de Microcontroladores
MCUTodo en uno vs Microprocesador
▼

Un microcontrolador (MCU) es un sistema computacional completo en un solo chip: incluye CPU, memoria RAM, memoria Flash (para el programa), y puertos de entrada/salida integrados. Es la diferencia entre una caja de herramientas completa vs. solo un martillo.

CaracterísticaMicroprocesador (CPU)Microcontrolador (MCU)
ComponentesSolo la CPU — necesita chips externosCPU + RAM + Flash + I/O en un chip
TamañoMás grande (requiere placa madre)Muy compacto — chip único
PotenciaAlta capacidad de procesamientoLimitado pero suficiente para su tarea
ConsumoAlto (vatios)Muy bajo (milivatios)
CostoAlto + periféricosMuy bajo ($1–$10)
AplicacionesPC, servidor, laptopArduino, electrodomésticos, IoT
🎯
Regla práctica: si necesitas procesar video, abrir múltiples aplicaciones o acceder a internet complejo → microprocesador. Si necesitas controlar un motor, leer un sensor o parpadear un LED de manera confiable y eficiente → microcontrolador.

Arquitectura interna de un microcontrolador
ArduinoPIC ESP32Periféricos integrados
▼

Dentro de un microcontrolador, todos los componentes conviven en un solo chip de silicio. Usan típicamente arquitectura Harvard para maximizar la velocidad con un bajo consumo energético.

🟢
Arduino (AVR / ARM)
Basado en ATmega328 (AVR) o ARM Cortex. 32KB Flash, 2KB RAM. Ideal para aprender electrónica y automatización.
🔴
PIC (Microchip)
Familia de microcontroladores muy populares en industria. Arquitectura RISC propia, muy robustos y con bajo consumo.
🔵
ESP32 (Espressif)
Dual-core ARM Cortex-M0, WiFi y Bluetooth integrados. El favorito para proyectos IoT por su conectividad y bajo costo.

Arquitecturas RISC y CISC
ARMx86 Instrucciones reducidas vs complejas
▼

Dos filosofías radicalmente distintas para diseñar el juego de instrucciones de un procesador. La pregunta central: ¿es mejor tener pocas instrucciones simples que se ejecutan rápido, o muchas instrucciones complejas que hacen más con menos código?

RISC — Reduced Instruction Set Computer
  • Pocas instrucciones, todas simples y de tamaño fijo
  • Cada instrucción se ejecuta en 1 ciclo de reloj
  • Diseño del hardware más simple y económico
  • Bajo consumo energético — ideal para móviles
  • Ejemplos: ARM (iPhone, Android), MIPS, RISC-V
CISC — Complex Instruction Set Computer
  • Muchas instrucciones, algunas muy complejas
  • Una instrucción puede realizar múltiples operaciones
  • Menos líneas de código ensamblador necesarias
  • Mayor consumo pero muy potente
  • Ejemplos: x86/x64 (Intel, AMD — todas las PCs)
💡
Hoy se usan los dos: los procesadores Intel/AMD (CISC x86) internamente dividen sus instrucciones complejas en micro-operaciones simples (como RISC). Y los procesadores ARM (RISC) de los nuevos Mac M1/M2/M3 son más rápidos que muchos x86 en eficiencia energética.
AspectoRISCCISC
Instrucciones~50–200 instrucciones300–1000+ instrucciones
Tamaño instrucciónFijo (32 bits)Variable (1–15 bytes en x86)
Ciclos por instrucción1 ciclo (pipeline eficiente)1–20+ ciclos
RegistrosMuchos registros generalesPocos registros especializados
Uso dominanteMóviles, IoT, Mac M-seriesPCs, servidores, laptops

// Módulo 12
Juego de Instrucciones (ISA)
Instruction Set Architecture LOAD STORE ADD Lenguaje ensamblador
▼

El ISA (Instruction Set Architecture) es el contrato entre el hardware y el software: define exactamente qué operaciones puede ejecutar un procesador, cómo se codifican en binario, y cómo accede a los datos. Es la "interfaz" que ven los compiladores y el sistema operativo.

// Tipos básicos de instrucciones
TipoInstrucciónAcciónAnalogía en Python
Transferencia LOAD R1, [addr] Copia dato de memoria al registro R1 x = lista[addr]
Transferencia STORE [addr], R1 Guarda R1 en memoria lista[addr] = x
Aritmética ADD R1, R2, R3 R1 = R2 + R3 a = b + c
Aritmética SUB R1, R2, R3 R1 = R2 − R3 a = b - c
Lógica AND R1, R2, R3 R1 = R2 AND R3 (bit a bit) a = b & c
Control CMP R1, R2 Compara R1 y R2, actualiza flags if a == b:
Control JMP [addr] Salta a otra instrucción (goto) continuar / break
Control CALL [func] Llama a una subrutina (función) funcion()
💡
Cuando escribes Python: el intérprete de Python traduce tu código a bytecode, que a su vez el sistema operativo convierte en instrucciones ISA del procesador (x86, ARM, etc.). Tu a = b + c se convierte en varios LOAD, ADD y STORE a nivel de hardware.
