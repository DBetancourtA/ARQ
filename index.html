¿Cómo entregar mis actividades?
// Guía de entrega en EVA · ARQ · Arquitectura de Computadoras
⚠️
Formato obligatorio: Redacta tus respuestas y diagramas desarrollados en clase. El archivo final debe entregarse en PDF.
// Pasos para la entrega
1
Crea un documento Word o Google Docs
Nómbralo exactamente así:
Apellido_Nombre_Practica_N  →  Ej: Torres_Diego_Practica_1.docx
2
Desarrolla el contenido solicitado
Escribe con tus propias palabras las respuestas a cada actividad. Incluye diagramas o esquemas donde se soliciten. Explica el razonamiento detrás de cada respuesta.
3
Agrega capturas de pantalla
Inserta capturas claras que evidencien tu trabajo: pantalla del archivo abierto, resultados obtenidos y cualquier configuración relevante.
4
Exporta el documento como PDF
Word: Archivo → Guardar como → PDF.
Google Docs: Archivo → Descargar → PDF.
Verifica que el PDF se vea bien antes de subirlo.
// Nombre del archivo final
Nombre del PDF
Apellido_Nombre_Practica_N.pdf
Ejemplo
Torres_Diego_Practica_1.pdf
Extensión requerida
.pdf  — no .docx, no .png, no .jpg
5
Sube el PDF a la plataforma EVA
Ingresa a EVA · ITB, busca la tarea de esta semana y sube el PDF antes de la fecha límite indicada.
// Verificación antes de entregar
✓
Nombre correcto: Apellido_Nombre_Practica_N.pdf
✓
El documento incluye capturas de pantalla claras del trabajo
✓
El archivo está en formato PDF — no Word, no imagen
✓
Subí el PDF a la tarea correcta en EVA antes del plazo

// Objetivo
Comprender cómo está organizado internamente un sistema computacional: procesador, memoria, buses y la relación entre hardware y software a nivel arquitectónico.
// Metodología
Cada semana incluye una página de referencia con los 12 temas desarrollados, diagramas SVG integrados y un taller por grupos con clave de acceso.
// Evaluación
Talleres grupales por semana, práctica de aula y exposición. El taller de grupos requiere clave de acceso asignada por el docente al inicio de cada sesión.
