Misión: CPU Humana
Simular el procesador como si fueran personas — dramatizar cómo se ejecuta una instrucción
🧠
// Tu misión paso a paso
1
Asigna roles a los miembros del grupo: Unidad de Control (jefe), ALU (calculadora humana), Memoria (archivador con papeles), Registro (pizarrón temporal)
2
Dramatiza la instrucción "Suma 5 + 3": Control pide a Memoria los números → Memoria los entrega → Control le da a ALU → ALU calcula → resultado va al Registro
3
Luego dramatiza "Guarda el resultado en Memoria": Control toma el valor del Registro → lo lleva al archivador → lo guarda en la dirección indicada
4
Explica qué es el Program Counter (PC): la persona que lleva la cuenta de qué instrucción sigue
5
Muestra el ciclo completo: Fetch (buscar instrucción) → Decode (entender) → Execute (ejecutar)
// Roles de actuación
🎩 Unidad de Control = jefe 🧮 ALU = calculadora 📂 Memoria = archivador 📋 Registro = pizarrón 🔢 PC = contador
Consejo para la actuación: usen objetos físicos — una calculadora real para la ALU, una carpeta para la Memoria, y papeles con números para simular los datos. Mientras más dramático, más memorable para la clase.
// Script de la dramatización
1️⃣
FETCH: El Jefe (CU) dice "necesito la instrucción #1" → va al archivador (Memoria) → saca el papel que dice "SUMA 5 + 3"
2️⃣
DECODE: El Jefe lee el papel y entiende: "hay que sumar. Los valores son 5 y 3. Los escribe en el pizarrón (Registros)"
3️⃣
EXECUTE: El Jefe le dice a la Calculadora (ALU): "suma lo que está en el pizarrón". La ALU dice "resultado: 8" y lo escribe en el pizarrón
4️⃣
STORE: El Jefe guarda el 8 en el archivador (Memoria) en la dirección indicada. El contador de instrucciones avanza al #2
🔁
Se repite para cada instrucción del programa. ¡Miles de millones de veces por segundo en una CPU real!
