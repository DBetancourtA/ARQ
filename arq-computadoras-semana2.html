// Concepto fundamental
¿Qué es un Bus de datos?
buscomunicaciónhardware
▾
Un bus es un conjunto de líneas de comunicación que conecta los componentes de una computadora — CPU, memoria y dispositivos I/O — y permite que los datos viajen entre ellos. Piénsalo como una autopista de datos dentro del computador.
⬡
Ancho de bus: indica cuántos bits viajan simultáneamente. Un bus de 64 bits transfiere el doble de datos por ciclo que uno de 32 bits. Cuanto más ancho el bus, mayor la capacidad de transferencia.
02
// Clasificación
Tipos de Bus: datos, dirección y control
bus de datosbus de direcciónbus de control
▾
El bus del sistema se divide en tres tipos según la función que cumplen. Cada uno transporta un tipo distinto de información entre la CPU y el resto del sistema.
Tipo de Bus¿Qué transporta?DirecciónBits típicos
Bus de Datos Los datos reales que se procesan (números, instrucciones, resultados) Bidireccional ↔ 8 / 16 / 32 / 64 bits
Bus de Dirección La ubicación de memoria donde leer o escribir datos Unidireccional → (CPU → Mem) 16 / 32 / 64 bits
Bus de Control Señales de gestión: leer, escribir, interrupciones, reloj Bidireccional ↔ 4 – 16 líneas
💡
Analogía: el Bus de Dirección es como la dirección de una casa. El Bus de Datos es el paquete que se entrega. El Bus de Control es el mensajero que indica si debe dejar o recoger el paquete.
⚠️
Bus de dirección determina la RAM máxima: con un bus de dirección de N bits, la CPU puede acceder a 2ᴺ posiciones de memoria. Un bus de 32 bits → máximo 4 GB de RAM (2³² = 4,294,967,296 bytes). Por eso los sistemas de 64 bits soportan terabytes de RAM.
03
// Estándares actuales
Buses modernos: PCIe, USB, SATA, NVMe
PCIeUSBSATANVMe
▾
Los buses modernos han reemplazado a los buses paralelos clásicos. Usan comunicación serial — menos cables pero a velocidades altísimas. Cada estándar tiene un uso específico.
⚡
PCIe
PCI Express. Conecta tarjetas de video, SSDs NVMe y accesorios de alto rendimiento. PCIe 5.0 alcanza 128 GB/s con 16 carriles.
🔌
USB
Universal Serial Bus. Periféricos externos: teclados, memorias, cámaras. USB 4.0 llega a 40 Gbps.
💾
SATA
Serial ATA. Conecta HDDs y SSDs SATA. Velocidad máxima 6 Gbps (SATA III). Más lento que NVMe.
🚀
NVMe
Non-Volatile Memory Express. SSDs directamente sobre PCIe. Gen 4 alcanza 7 GB/s. El más rápido para almacenamiento.
🌐
Thunderbolt
Desarrollado por Intel + Apple. Combina PCIe y DisplayPort. Thunderbolt 4 ofrece 40 Gbps bidireccionales.
📡
DMI / HyperTransport
Buses internos que conectan la CPU con el chipset (Intel: DMI, AMD: Infinity Fabric). No visibles al usuario.
📌
Bus serie vs. Bus paralelo: los buses antiguos eran paralelos (muchos cables simultáneos), pero a altas frecuencias el ruido eléctrico entre cables causaba errores. Los buses serie modernos usan pocos cables pero a frecuencias altísimas, logrando mayor velocidad y confiabilidad.

// Concepto fundamental
¿Qué es la memoria de una computadora?
memoriaalmacenamiento temporalbytes
▾
La memoria es el espacio donde la computadora almacena temporalmente los datos e instrucciones que está procesando en este momento. Sin memoria, la CPU no tendría con qué trabajar — sería como un cerebro sin recuerdos de corto plazo.
⬡
Diferencia clave: la memoria (RAM) es temporal y volátil — pierde todo al apagarse. El almacenamiento (HDD/SSD) es permanente y no volátil — conserva los datos aunque no haya energía.

Jerarquía de memoria — Pirámide de velocidad

▾
La memoria no es un sistema único — existe una jerarquía: más arriba es más rápida y cara, pero más pequeña. Más abajo es más lenta y barata, pero con mayor capacidad. La CPU siempre busca los datos empezando desde arriba.

⚠️
Principio de localidad: los programas tienden a acceder repetidamente a las mismas zonas de memoria (localidad temporal) y a posiciones cercanas entre sí (localidad espacial). La caché explota este principio para anticipar qué datos necesitará la CPU.

// Tipos de memoria principal
RAM vs ROM — Volátil y No Volátil
RAMROMEEPROMFlash
▾
RAM (Random Access Memory) es volátil: almacena temporalmente lo que la CPU está usando ahora mismo. ROM (Read-Only Memory) es no volátil: contiene instrucciones permanentes grabadas de fábrica, como el firmware de arranque (BIOS/UEFI).
CaracterísticaRAMROM
VolatilidadVolátil — pierde datos al apagarseNo volátil — persiste sin energía
Lectura/EscrituraLectura y escritura en tiempo realSolo lectura (o escritura muy lenta)
VelocidadMuy rápida (nanosegundos)Más lenta que RAM
Uso típicoProgramas en ejecución, datos activosBIOS, firmware, microcódigo
Capacidad típica4 GB – 128 GB en PCsMegabytes (firmware)
// Variantes de ROM moderna
📀
PROM
Programmable ROM. Se programa una sola vez con un dispositivo especial. No se puede modificar después.
🔄
EPROM
Erasable PROM. Se borra con luz ultravioleta y se reprograma. Identificable por su ventana de cuarzo.
⚡
EEPROM
Electrically Erasable PROM. Se borra y reprograma eléctricamente. Base del almacenamiento Flash actual.
🚀
Flash (NAND)
Evolución de EEPROM. Rápida, barata y compacta. Usada en SSDs, USB drives, tarjetas SD y smartphones.

// Tecnología interna
SRAM vs DRAM — Caché vs Memoria principal
SRAMDRAMcachérefresco
▾
La RAM no es un solo tipo de tecnología. SRAM y DRAM se construyen de manera diferente, lo que les da características opuestas: velocidad vs. capacidad vs. costo.
CaracterísticaSRAM (Static)DRAM (Dynamic)
TecnologíaFlip-flops (6 transistores/bit)Capacitor + transistor (1 transistor/bit)
VelocidadMuy rápida (~1 ns)Más lenta (~10–100 ns)
RefrescoNo necesita refrescoNecesita refresco periódico (cada ~64 ms)
DensidadBaja (ocupa más espacio)Alta (más bits por chip)
CostoMuy caraMás económica
Uso típicoCaché L1, L2, L3 dentro de la CPURAM del sistema (módulos DIMM)
⬡
¿Por qué DRAM necesita refresco? Los capacitores que almacenan cada bit pierden su carga eléctrica con el tiempo. El controlador de memoria debe "releer y reescribir" constantemente los datos para que no se pierdan — eso es el refresco dinámico.

// Historia y evolución
Evolución de la RAM: DDR1 → DDR5
DDRSDRAMvelocidadvoltaje
▾
DDR significa Double Data Rate — transfiere datos dos veces por ciclo de reloj (en el flanco de subida y bajada). Cada generación duplica aproximadamente el ancho de banda y reduce el voltaje para mayor eficiencia energética.
DDR1
1998 – 2004
⚡ 200–400 MT/s
📦 64–512 MB típico
🔋 2.5 V
📌 184 pines (DIMM)
Primera generación DDR. Reemplazó a SDRAM. Ya obsoleta.
DDR2
2003 – 2008
⚡ 400–1066 MT/s
📦 512 MB – 4 GB
🔋 1.8 V
📌 240 pines (DIMM)
Prefetch de 4 bits. Menor voltaje, mayor densidad que DDR1.
DDR3
2007 – 2014
⚡ 800–2133 MT/s
📦 1–16 GB por módulo
🔋 1.5 V (1.35 V LP)
📌 240 pines (DIMM)
Prefetch de 8 bits. Estándar dominante por casi una década.
DDR4
2014 – 2022
⚡ 1600–3200 MT/s
📦 4–32 GB por módulo
🔋 1.2 V
📌 288 pines (DIMM)
Banco de 16 grupos. Mayor fiabilidad y XMP overclock.
DDR5
2020 – Presente
⚡ 4800–8400+ MT/s
📦 16–128 GB por módulo
🔋 1.1 V
📌 288 pines (nuevo notch)
2 canales independientes por DIMM. ECC integrado en el módulo.
📊
MT/s vs MHz: la velocidad de RAM se expresaba en MHz (frecuencia efectiva) pero actualmente se usa MT/s (megatransferencias por segundo) porque DDR transfiere datos dos veces por ciclo. DDR4-3200 trabaja a 1600 MHz físicos pero hace 3200 millones de transferencias por segundo.
⚠️
Incompatibilidad entre generaciones: DDR1, DDR2, DDR3, DDR4 y DDR5 tienen conectores físicamente incompatibles — el número de pines y la posición de la muesca cambian en cada generación para evitar instalarlos incorrectamente. No se puede usar un módulo DDR4 en una placa madre DDR5.

// Optimización de rendimiento
Memoria Caché — El buffer de alta velocidad de la CPU
caché L1caché L2caché L3hit/miss
▾
La caché es una pequeña cantidad de memoria SRAM ultra-rápida integrada dentro del chip de la CPU. Su función es guardar las instrucciones y datos que la CPU usa con más frecuencia, evitando el costoso viaje hasta la RAM principal.
NivelUbicaciónTamaño típicoLatenciaCompartida
L1 Dentro de cada núcleo 32–64 KB por núcleo ~1–4 ciclos No (privada)
L2 Dentro de cada núcleo 256 KB – 2 MB por núcleo ~12 ciclos No (privada)
L3 Compartida entre todos los núcleos 4–64 MB (o más) ~30–40 ciclos Sí (todos los núcleos)
// Cache Hit y Cache Miss
✅
Cache HIT
Los datos que necesita la CPU están en la caché. Se obtienen en pocos ciclos. El rendimiento es máximo. En programas bien optimizados, el hit rate supera el 95%.
❌
Cache MISS
Los datos no están en caché. La CPU debe ir a buscarlos en RAM (o peor, en disco). Puede suponer esperar 100+ ciclos. Es el principal cuello de botella de rendimiento.

// Sistema operativo + hardware
Memoria Virtual — Más RAM de la que tienes físicamente
memoria virtualpaginaciónswapMMU
▾
La memoria virtual es una técnica que permite a los programas usar más memoria de la que hay físicamente instalada. El sistema operativo utiliza una porción del disco duro como extensión de la RAM — llamada archivo de intercambio (swap).
🗺️
Paginación
La memoria se divide en bloques de tamaño fijo llamados páginas (típicamente 4 KB). La MMU traduce las direcciones virtuales a físicas en tiempo real.
🔄
Swap / Page file
Cuando la RAM se llena, el SO mueve páginas menos usadas al disco. Esto es mucho más lento pero evita que el sistema se bloquee.
🧭
MMU
Memory Management Unit. Chip (integrado en la CPU) que realiza la traducción de direcciones virtuales a físicas automáticamente en cada acceso a memoria.
🛡️
Protección de memoria
La memoria virtual también aísla los espacios de cada proceso. Un programa no puede acceder a la memoria de otro — esto previene crashes y ataques de seguridad.
⚠️
Thrashing: cuando el sistema usa constantemente el swap porque la RAM es insuficiente, el rendimiento cae drásticamente. El disco funciona a 1000x más lento que la RAM. La solución es agregar más RAM física, no más swap.
💡
En la práctica: Windows usa el archivo pagefile.sys como swap. Linux usa una partición dedicada swap o un archivo swapfile. macOS comprime primero la memoria antes de usar swap (memoria comprimida).
